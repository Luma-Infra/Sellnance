# Sellnance 기능 개선 및 버그 수정 통합 계획

차트 전체화면 툴바 수정, 레이아웃 복원 버그 해결, 방향키 코인 탐색 순서 수정, 탭 컨테이너 UX 통일에 더해 **데이터 정렬 시 값 없는 하이픈(-) 항목만 최하단에 배치하는 정렬 개선**을 하나의 통합 계획으로 수립합니다.

## 제안된 변경 사항

### 1. 데이터 정렬 개선 (값 없는 하이픈만 최하단 배치)

#### [MODIFY] [table_sort.js](file:///c:/Users/78831/Sellnance/static/table_sort.js)
- 수치가 `0`인 데이터는 정상적으로 정렬 범위에 포함하고, 데이터가 아예 존재하지 않아 화면에 하이픈(`-`)으로 표시되는 항목(null, undefined이거나 포맷팅 텍스트가 `"-"`인 경우)만 오름차순/내림차순 상관없이 항상 최하단에 강제 배치합니다.
- `simpleSortData` 내 `isEmpty` 감지 로직을 다음과 같이 세분화합니다:
  - **U-Vol (VolumeUpbit)**: `val`이 null/undefined/"-" 이거나 `d.Upbit_Vol_Formatted === "-"` 인 경우 (0은 정상 수치로 정렬)
  - **Volume**: `val`이 null/undefined/"-" 이거나 `d.Volume_Formatted === "-"` 인 경우
  - **MarketCap**: `val`이 null/undefined/"-" 이거나 `d.MarketCap_Formatted === "-"` 인 경우
  - **VMC**: `val`이 null/undefined/"-" 이거나 `d.VMC_Formatted === "-"` 인 경우
  - **Funding**: `val`이 null/undefined/"-" 이거나 `d.Funding_Formatted === "-"` 인 경우
  - **Kimchi (김프)**: `d.Kimchi_Label === "-"` 이거나 비어있는 경우
- 화면상의 모든 누락 값은 UI 렌더러(`table_render.js`)에 의해 하이픈(`-`)으로 일관되게 표시되므로, 이를 기준으로 정확히 판단합니다.
