# 빗썸/업비트 1분봉 ~ 월봉 매핑 및 조립 정합성 개선 계획

빗썸 API의 제한적인 인터벌 지원과 `mainStep` 조립 누락 문제를 해결하여 과거 김프 지표 누락 및 튐 현상을 해결합니다.

## User Review Required
> [!IMPORTANT]
> - 빗썸 API는 `15m` 인터벌을 자체 지원하지 않으므로, `5m` API 데이터를 가져온 후 3개씩 묶어 `15m` 봉으로 조립합니다.
> - `12h` 봉은 빗썸 API가 `12h`를 네이티브 지원하므로 조립 없이 그대로 사용하며, `3d`, `1w`, `1M` 등의 다일 봉은 `24h` (일봉)을 가져온 후 각각 3개, 7개, 월단위 기준 날짜로 병합(조립) 처리합니다.
> - 업비트와 동일하게 빗썸 데이터 처리부에도 `mainStep` 루프를 적용하여 통일된 OHLCV 조립 메커니즘을 갖춥니다.

## Proposed Changes

- 김프용 서브 데이터 조회 시에도 위와 동일한 조립 로직을 통과시켜 `calculateKimchiData`에 전달되는 `subRaw`의 타임스탬프 정합성을 확보합니다.
