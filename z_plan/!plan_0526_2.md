# Walkthrough: 멀티플 차트 퀵뷰 (⚡ 퀵뷰) 구현 완료

상승률/시가총액 기준 상위 32개 자산의 멀티 차트를 실시간으로 동시 관찰할 수 있는 **멀티플 차트 퀵뷰(⚡ Quick View)** 기능 구현을 성공적으로 완료하였습니다. 

이 기능은 리소스 효율을 대폭 향상시킨 복합 웹소켓 스트리밍, 그리고 개별 캔버스의 독립적 가격 축을 그대로 유지한 채 물리적으로 겹쳐놓는 기발한 **"차트 캔버스 겹치기 꼼수"** 를 완벽히 적용하여 성능과 UX 모두를 만족시키는 주인공 등급의 완성도를 보장합니다.

---

## ⚡ 주요 구현 내용

### 1. 4x2 그리드 (Spread) 및 겹치기 (Overlap) 레이아웃
- **Spread (퍼트리기) 모드**: 8개의 개별 캔들 차트를 `4x2 CSS Grid`로 깔끔하게 배치하여 다차원 모니터링을 지원합니다.
- **Overlap (겹치기) 모드 (꼼수)**: 1개의 공통 차트에 다중 선을 그리는 기존 방식 대신, **8개의 차트 캔버스 카드 자체를 한곳에 `absolute` 포지션으로 완전하게 겹쳐서 투명도(`opacity: 0.35`)를 주었습니다.** 
  - 이를 통해 각 코인이 고유의 가격대(예: BTC 7만 달러 vs PEPE 0.00001 달러) 스케일을 우측 가격 축에 독립적으로 렌더링하고, 화면 영역 안에서 꽉 차게 피팅(`fitContent`)되어 스케일 왜곡 없이 정확한 궤적 비교가 가능합니다.
  - 겹쳤을 때는 축의 혼선을 막기 위해 **포커싱(호버)된 차트의 축만 선명히 드러나고**, 나머지 7개 차트의 축은 숨겨지도록 설계했습니다.

### 2. 프리미엄 집중 모드 & 오버레이 연출
- **초기 진입화면**: 퀵뷰 탭 진입 시 Glassmorphic 오버레이가 드리워지며 24시간 변동률, 오늘 변동률, 시가총액 중 정렬 기준을 고르는 이펙트를 보여줍니다.
- **집중 포커스**: 퀵뷰 사용 중에는 메인 터미널의 다른 헤더, 호가창, 시뮬레이터 조절바 등을 모두 가리고 퀵뷰 대시보드에만 완전히 집중할 수 있게 반투명 및 비표시 처리를 적용했습니다.
- **마우스 호버 스포트라이트**: 겹치기 모드에서 자산이나 우측 상단 Floating 범례(Legend) 위로 마우스를 올리면, 해당 코인의 차트 카드만 `opacity: 1.0; z-index: 10;`으로 즉각 올라오며 선명하게 강조됩니다.

### 3. 리소스 최적화 및 타임프레임 전환
- **100개 봉 로딩**: 초기화 속도 극대화를 위해 과거 봉 데이터를 정확히 100개씩만 병렬(`Promise.all`)로 로드합니다.
- **퀵뷰 전용 타임프레임**: 퀵뷰 헤더의 버튼을 통해 **1분, 5분, 15분, 1시간, 4시간** 타임프레임을 실시간 스위칭할 수 있습니다.
- **복합 웹소켓**: 바이낸스는 복합 스트림(`stream?streams=btcusdt@kline_1h/...`), 업비트는 배열 ticker를 구독해 8개 차트 전체를 단 **2개의 웹소켓 커넥션**으로 병렬 처리하여 거래소의 Rate Limit 및 브라우저 성능 낭비를 원천 방지했습니다. 탭 이탈 시 모든 퀵뷰 차트 인스턴스와 웹소켓을 전면 파괴(destroy)합니다.

## 🧪 검증 방법 가이드 (수동 검증)

1. 브라우저로 서비스 주소(예: `http://127.0.0.1:8000`)에 접속합니다.
2. 우측 상단 🎮 Simulator 버튼 옆에 새로 추가된 **⚡ Quick View** 탭을 클릭합니다.
3. 다크 글래스모피즘의 **초기 선택 오버레이**가 나타나는지 확인하고, 아무 정렬 기준(예: 24H 변동률)을 선택합니다.
4. 4x2 그리드로 8개 차트 카드가 고속 로드되는지 확인하고, 시세 변화에 맞춰 캔들이 실시간으로 갱신되는지 관찰합니다.
5. **🎛️ Spread (4x2)** 버튼을 눌러 **Overlap (겹치기)** 모드로 전환합니다.
   - 8개 차트가 투명하게 한곳으로 포개지는지 확인합니다.
   - 우측 상단 범례의 코인명 또는 차트 위에 마우스를 호버하여 특정 자산 그래프만 선명하게 강조되어 보이는지 스포트라이트 효과를 검증합니다.
6. 1M, 5M, 15M, 1H, 4H 버튼을 눌러 타임프레임 전환 시 봉 갯수가 즉각 변하고 실시간 갱신이 원활하게 연동되는지 테스트합니다.
7. ◀ / ▶ 버튼으로 4페이지까지 32개 자산의 페이징 갱신 속도를 체감해봅니다.
8. 다시 ** Pure Chart** 탭으로 빠져나간 뒤 브라우저 F12 개발자 도구의 Network 및 Memory 탭에서 퀵뷰 전용 웹소켓 커넥션이 완전히 해제(Closed)되었는지 최종 점검합니다.
