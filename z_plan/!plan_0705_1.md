# 구현 계획서 - 김치 프리미엄 계산 시 stale(오래된) 국내 가격 참조 버그 수정

## 목표 설명

USDT 선물/현물 전용 티커(예: `ONGUSDT`)의 김치 프리미엄이 실시간 디버그 로그에서 갑자기 1%p 이상 급변(예: `-1.19% ➔ 2.82%`)하는 버그를 해결합니다.

**원인 분석:**
[stream_table.js](file:///c:/Users/78831/Sellnance/static/stream_table.js) 내 `calcKimchi` 함수는 김프 계산 시 타겟 행의 자체 국내 가격(`r.Upbit_Price` 또는 `r.Bithumb_Price`)을 최우선으로 사용합니다. 
하지만 USDT 전용 행의 경우, 페이지가 최초 로드될 때 백엔드로부터 전달받은 국내 가격(예: `77.26`원)으로 초기화된 이후 실시간 국내 소켓에 의해 업데이트되지 않아 stale(오래된) 상태로 유지됩니다.

이 stale한 가격을 기반으로 계산을 지속하다가, 5분 주기로 동작하는 백그라운드 동기화 함수(`loadTableDataSilent`)가 실행되면 해당 행의 국내 가격 정보가 최신(예: `80.1`원)으로 갱신되면서 김프가 갑자기 크게 튀는 현상이 발생합니다.

**해결 방안:**
실시간 소켓을 통해 정상적으로 국내 가격이 계속 갱신되는 원화 마켓용 쌍둥이 행(`krwRow`)의 가격을 최우선 순위로 가져오도록 `calcKimchi` 내의 국내가 선택 로직(`priceKor`)의 우선순위를 조정합니다.

## 변경 사항

### static

#### [MODIFY] [stream_table.js](file:///c:/Users/78831/Sellnance/static/stream_table.js)
`store.uidToKrwRowMap`을 통해 얻은 `krwRow`의 실시간 원화 가격을 r.Upbit_Price보다 우선 참조하도록 변경합니다.

```javascript
// 기존 코드
if (hasUpbit) {
  priceKor = r.Upbit_Price || (krwRow ? krwRow.Upbit_Price || krwRow.Price_KRW : 0) || r.Price_KRW || 0;
} else if (hasBithumb) {
  priceKor = r.Bithumb_Price || (krwRow ? krwRow.Bithumb_Price || krwRow.Price_KRW : 0) || r.Price_KRW || 0;
}

// 변경 코드
if (hasUpbit) {
  priceKor = (krwRow ? krwRow.Upbit_Price || krwRow.Price_KRW : 0) || r.Upbit_Price || r.Price_KRW || 0;
} else if (hasBithumb) {
  priceKor = (krwRow ? krwRow.Bithumb_Price || krwRow.Price_KRW : 0) || r.Bithumb_Price || r.Price_KRW || 0;
}
```

## 검증 계획

### 수동 검증
1. 로컬 환경에서 웹 어플리케이션을 구동합니다.
2. 업비트와 바이낸스 선물에 동시에 상장된 코인(예: `ONGUSDT`)을 디버그 패널에서 모니터링합니다.
3. 국내 원화 마켓 가격 변동 시, 5분 주기의 백그라운드 동기화를 기다리지 않고도 김치 프리미엄이 실시간으로 부드럽게 연동되어 반영되는지 확인합니다.
