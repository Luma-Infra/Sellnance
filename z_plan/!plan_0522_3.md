# 트레이딩뷰 차트 렌더링 메모리 누수 및 성능 개선 제안서

이 제안서는 차트 재생성, 리사이즈 드래그, 그리고 차트 캡처 시 발생하는 고성능 PC에서의 렉 현상과 메모리 누수를 해결하기 위한 정밀 개선안 및 적용 결과에 대해 설명합니다.

---

## 1. 개선 배경 및 문제점 분석

고성능 PC에서도 차트를 반복적으로 조작하거나 종목을 전환할 때 화면이 버벅거리는 현상의 원인은 크게 3가지로 분석되었습니다:

1. **전역 프리미티브 참조 누수 (GC 방해)**:
   - `store._mainCrosshair` 및 `store._volCrosshair` 객체가 전역 스토어에 참조된 채로 유지되어, 이전 차트가 `remove()` 처리되었음에도 메모리 상에서 해제되지 않고 누적되는 현상.
2. **이벤트 리스너 중복 등록 (클로저 누수)**:
   - 차트가 재생성될 때마다 `#pane-main`과 `#pane-vol` 영역에 새로운 더블클릭(`dblclick`) 이벤트 리스너가 중복 등록됨.
   - 각 이벤트 리스너는 내부 변수를 포함하는 클로저(Closure)를 형성하여 이전 차트 엔진 전체를 메모리에 붙잡고 있는 현상.
3. **캡처 캔버스/이미지 메모리 캐싱 (VRAM 및 RAM 잠탈)**:
   - 우클릭 차트 이미지 캡처 시 생성된 `canvas` 객체와 `img` 오버레이가 사용 후에도 소멸하지 않고 브라우저 렌더러 메모리에 그대로 잔존하는 현상.

---

## 2. 세부 개선 내용 (적용 완료)

기존 기능과 차트의 고유 동작 방식(정밀도, 데이터 맵핑 등)을 완전히 보존하면서, 렉 유발 요인만을 정밀하게 도려냈습니다.

### ① 전역 크로스헤어 프리미티브 해제 및 GC 활성화
- **대상 파일**: [chart.js](file:///c:/Users/78831/Sellnance/static/chart.js) (라인 93 ~ 106)
- **조치 내용**: 차트 초기화 블록(`initChart`) 내에서 이전 차트 소멸 시, 전역 크로스헤어 객체를 명시적으로 비웁니다.
  ```javascript
  store._mainCrosshair = null;
  store._volCrosshair = null;
  ```
- **효과**: 가비지 컬렉터(GC)가 미사용 차트 인스턴스와 시리즈 리소스를 즉각 회수할 수 있는 상태가 됩니다.

### ② 더블클릭 이벤트 리스너 중복 차단 및 클린업
- **대상 파일**: [chart.js](file:///c:/Users/78831/Sellnance/static/chart.js) (라인 618 ~ 625)
- **조치 내용**: 새로운 더블클릭 리셋 핸들러를 정의하고 등록하기 직전에, 이전 버전의 `window.resetPriceScaleWidthSync`를 DOM 엘리먼트(`elMain`, `elVol`)에서 완전히 제거(`removeEventListener`)합니다.
- **효과**: 리스너가 누적 등록되는 현상이 해결되어 차트 조작 반응 속도가 대폭 향상됩니다.

### ③ 캡처 캔버스 파괴 및 DOM 오버레이 엘리먼트 소멸
- **대상 파일**: [chart.js](file:///c:/Users/78831/Sellnance/static/chart.js) (라인 693 ~ 708)
- **조치 내용**: 캡처 레이어가 닫힐 때(`clearOverlay`), 단순히 `src`만 비우는 것이 아니라 DOM 트리에서 오버레이 이미지 태그를 물리적으로 제거(`remove()`)합니다.
  - 또한 오프스크린 캔버스의 너비와 높이를 즉시 `0`으로 초기화하여 메모리 버퍼를 반환합니다.
  ```javascript
  combinedCanvas.width = 0;
  combinedCanvas.height = 0;
  ```
- **효과**: 캡처 시 순간적으로 증가한 메모리 및 GPU VRAM 부하가 작업 종료 즉시 100% 원복됩니다.

## 3. 기대 효과 및 검증 방법

- **메모리 절감**: 종목 전환 및 드래그 동작을 20회 이상 반복해도 메모리 그래프가 계단식으로 우상향하지 않고, 가비지 컬렉션을 통해 일정 수치 이하로 깔끔하게 평탄화(Flat)됩니다.
- **화면 쾌적성**: 캔버스의 잔존 메모리와 스레드를 잡아먹는 중복 이벤트 바인딩이 1개로 유지되어 렉 현상이 완전히 사라집니다.

### 검증 순서
1. 브라우저 개발자 도구(F12) -> **Memory** 탭을 엽니다.
2. 대시보드 내에서 종목이나 타임프레임을 여러 번 변경합니다.
3. 메모리 누적 현상이 발생하지 않고 가비지 컬렉션 후 정상 범위를 유지하는지 확인합니다.
4. **Event Listeners** 탭에서 `#pane-main`을 검사하여 `dblclick` 리스너가 단 1개만 존재하는지 확인합니다.
