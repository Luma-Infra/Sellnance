# 📊 Sellnance 런타임 성능 개선 및 렉 해소 구현 계획서 (Performance Plan)

본 문서는 Chrome DevTools 프로파일러 데이터 분석에 근거하여 실시간 웹소켓 이벤트(`aggTrade`) 유입 시 발생하는 프레임 드랍, 브라우저 레이아웃 무효화(InvalidateLayout), V8 엔진 가비지 컬렉션(GC) 폭주 문제를 해결하기 위한 구체적인 수정 범위와 코드를 정의합니다.

## User Review Required

> [!IMPORTANT]
> 1. **상단 OHLC 패널의 레이아웃 격리**: 주변 하위 테이블의 리플로우(Reflow) 연산 전파를 완전히 차단하기 위해 CSS `contain` 속성을 적용합니다.
> 2. **웹소켓 데이터 실시간 배치 버퍼링 도입**: 유입 즉시 DOM을 갱신하는 것이 아닌, `requestAnimationFrame` 주기에 맞춘 일괄 배치 업데이트(Batch Update) 구조로 전환합니다.
> 3. **Vite 빌드 및 배포**: 최종 변경 코드는 `npx vite build`를 통해 `dist/` 빌드 자산으로 갱신되어 배포됩니다.

## Proposed Changes

### [CSS 스타일시트 격리]

#### [MODIFY] [z_style.css](file:///c:/Users/kmj/Sellnance/static/z_style.css)
* **목적**: 상단 OHLC 값 변경이 하위 테이블 영역 전체의 스타일 및 레이아웃을 다시 계산하게 만드는(Layout Thrashing) 현상을 방어하기 위해 CSS 격리 바운더리를 지정합니다.
* **상세 작업**: 
  * `#ohlc-legend` 및 `#ohlc-values`를 감싸는 최상위 부모 컨테이너에 고정 높이/너비(혹은 flex 구조 내 최소크기)를 준 뒤 `contain: layout paint size;` 혹은 `contain: strict;` 속성을 주입합니다.

### [Javascript 데이터 핸들러 및 렌더링 엔진]

#### [MODIFY] [stream.js](file:///c:/Users/kmj/Sellnance/static/stream.js)
* **목적**: 웹소켓 데이터 수신 즉시 개별적으로 DOM 셀을 직접 조작하던 비효율적인 동기 업데이트 루프를 끊어내고 가비지 컬렉션(GC) 생성을 억제합니다.
* **상세 작업**:
  * `Bt.i.sniperWs.onmessage` 수신 즉시 `updateRowPriceDisplay`를 다이렉트로 호출하지 않고, 새로 수신된 수치 데이터를 메모리 상의 전역 버퍼 객체(`store.priceUpdateBuffer`)에 누적시킵니다.
  * `requestAnimationFrame` 루프를 가동하여 매 프레임(약 16.6ms)마다 버퍼에 데이터가 쌓여 있을 때만 단 한 번에 일괄적으로 변경이 필요한 화면 셀을 갱신(`Batch Update`)합니다.

#### [MODIFY] [chart.js](file:///c:/Users/kmj/Sellnance/static/chart.js)
* **목적**: 십자선 마우스 움직임 및 화면 줌/스크롤 시 차트 내부 Y축 너비 계산 함수(`syncPriceScaleWidths`)가 지나치게 잦은 주기로 호출되어 레이아웃 재계산을 자극하는 현상을 방어합니다.
* **상세 작업**:
  * 마우스 무브 등 드래그/인터랙션 연산 중에는 가격 스케일 정밀 동기화 실행 주기를 150ms 단위의 스로틀러(Throttler) 락을 통해 디바운싱 처리합니다.
