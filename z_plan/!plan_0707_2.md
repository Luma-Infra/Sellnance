# 구현 계획서 - 퀵뷰 거래소 매칭 규칙 단일화 및 실시간 동기화 개선

이 계획서는 퀵뷰(QuickView) 차트 카드들의 거래소 데이터 매칭 우선순위를 사용자 요청에 맞게 단순화하고, 이에 맞춰 실시간 웹소켓(Websocket) 및 헤더 가격 표시 방식을 전면 동기화합니다.

## 기존 문제점
- 기존 퀵뷰는 업비트 상장 코인일 경우 국내외 구분 없이 무조건 업비트 데이터를 우선 로드했습니다.
- 해외 전용 코인의 경우 퀵뷰 마켓 필터(`store.qvMarket`) 상태에 따라 복잡한 우선순위 규칙이 작동했습니다.
- 이로 인해 국내외 교차 상장 코인들이나 복수 거래소 상장 자산들의 매칭 흐름이 직관적이지 않았습니다.

## 제안된 변경 사항

### 거래소 우선 매칭 규칙 단순화 (Unification)
국내외 구분 없이 다음 우선순위로 데이터를 조회하고 바인딩합니다:
1. **바이낸스 선물 (`binance_futures`)** (상장 리스트에 있을 경우)
2. **바이낸스 현물 (`binance_spot`)** (상장 리스트에 있을 경우)
3. **업비트 (`upbit`)** (업비트에 상장되어 있을 경우)
4. **빗썸 (`bithumb`)** (빗썸에 상장되어 있을 경우)

- `updateLiveHeaderPrice()` 함수에서 `resolvedExchange`가 국내 거래소(`upbit`, `bithumb`)일 때는 원화(₩)로, 해외 거래소일 때는 달러($)로 가격 단위를 통일하여 표시합니다.

## 검증 계획

### 수동 검증
1. **퀵뷰 거래소 헤더 매칭 확인**:
   - 퀵뷰 창을 열고, 국내외 동시 상장 코인(예: BTC, ETH 등)의 차트 헤더가 **BINANCE FUTURES**로 우선 지정되었는지 확인합니다.
   - 바이낸스에 없는 국내 전용 코인의 경우 **UPBIT** 또는 **BITHUMB**으로 올바르게 지정되는지 확인합니다.
2. **실시간 데이터 수신 테스트**:
   - 퀵뷰가 켜진 상태에서 각 차트의 실시간 가격 등락 및 캔들 업데이트가 원활히 작동하는지 확인합니다.
3. **배포 빌드 검증**:
   - `npx vite build`를 실행하여 컴파일 오류가 없는지 다시 확인합니다.
