# 구현 계획서 - 테이블 실시간 갱신 중단 및 특정 Row 겹침/비어버림 UI 버그 수정

## Goal Description
다음 UI 결함들을 수정합니다.

1. **코인 클릭 시 실시간 업데이트 및 정렬(경주마) 중단 오류:**
   - **원인:** 특정 코인을 클릭(선택)할 때 `ui_control.js` 내 `selectSymbol`이 전역 마켓 모드인 `store.currentMarket`을 해당 코인의 거래소 마켓(예: `"UPBIT"`)으로 강제 변경합니다. 이 때문에 테이블 렌더링 및 틱 필터링 로직들이 선물 모드(`FUTURES`)를 이탈하여 바이낸스 선물 틱의 갱신이 원천 차단됩니다.
   - **해결책:** **"테이블 뷰 기준의 활성 마켓"**(`store.currentMarket`)과 **"차트/호가창 등 상세 뷰 기준의 활성 마켓"**(`store.currentChartMarket`)을 분리하여 개별 코인 선택이 테이블 정렬 및 갱신 주기에 영향을 주지 않도록 격리합니다.

2. **31등에 코인이 쌓이고 33등, 36등 등 특정 Row가 비어버리는 오류:**
   - **원인:** `table_render.js`의 `renderTable(isRealtime)` 내에서 실시간 갱신 시 위치 재배치 조건식인 `needsPositionUpdate`가 30등 밖의 코인들에 대해서는 업데이트를 생략하도록 막고 있습니다.
   - **해결책:** 실시간 정렬 여부와 관계없이 순위 인덱스가 달라졌다면(`oldIndex !== i`) 무조건 위치 재배치와 등수 갱신을 하도록 방어 조건을 추가합니다.

3. **AHT 등 업비트/빗썸 전용 코인 클릭 시 우측 헤더 정보(가격/변동률) 미노출 오류:**
   - **원인:** 우측 상세 대시보드를 갱신하는 `_main.js` 내 `updateHeaderDisplay` 함수가 활성 마켓을 판단할 때, 개별 선택된 코인의 마켓 정보인 `store.currentChartMarket`이 아닌 테이블 뷰 기준의 `store.currentMarket`을 그대로 사용하고 있습니다. 
     예를 들어, 테이블 탭이 `"BINANCE"`인 상태에서 업비트 전용 코인을 클릭하면, 대시보드가 이 코인을 바이낸스 가격 기준으로 가져오려 하여 가격과 변동률이 모두 `0.0000 원` / `0.00%`로 노출되는 현상이 발생합니다.
   - **해결책:** `updateHeaderDisplay` 내의 활성 마켓 및 선물 판단 기준을 `store.currentMarket`에서 격리된 **`store.currentChartMarket`**으로 변경합니다.

## Verification Plan

### Manual Verification
1. 테이블 상단 마켓 필터 탭에서 **"BINANCE"** 탭을 클릭하여 활성화합니다.
2. 테이블 목록에서 업비트/빗썸 전용 코인인 **"AHT" (아하토큰)** 또는 **"MED" (메디블록)**과 같은 코인의 Row를 클릭합니다.
3. 우측 대시보드 헤더 영역의 정보들을 검증합니다:
   - 가격(Price)이 `0.0000`이 아닌 실제 업비트 시세에 맞는 금액(원화 및 달러)으로 정상 표시되는지 확인합니다.
   - 24H Chg 및 Day Chg 변동률이 `0.00%`가 아닌 정상적인 전일비/당일비 수치로 표시되는지 확인합니다.
   - 시가총액(Market Cap) 및 거래대금(Vol, U-Vol)도 정상적으로 표시되는지 확인합니다.
