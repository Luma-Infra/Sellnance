# API 키 보안 누출 방지 패치 구현 계획서

서버의 CMC API 키 원본이 웹페이지 개발자 도구(F12)나 API 조회를 통해 유출되는 보안 취약점을 완전히 제거하고, 키의 존재 여부(exists)만 브라우저가 확인하여 흐름을 제어하도록 패치합니다.

---

## 제안된 변경 사항

### [컴포넌트] API 보안 및 초기 기동 레이어

#### [수정] [app.py](file:///c:/Users/78831/Sellnance/modules/app.py)
* `@app.get("/api/get-env-key")` 함수에서 `return {"key": env_key}` 대신 **`return {"exists": env_key != ""}`**로 변경하여 브라우저로의 키 원본 유출을 원천 봉쇄합니다.

#### [수정] [start.js](file:///c:/Users/78831/Sellnance/static/start.js)
* `initStartScreen()` 함수에서 `data.key` 대신 `data.exists` 조건식을 확인하도록 변경합니다.
* 서버 키가 존재할 시, 브라우저가 키 원본을 알 필요가 없으므로 `localStorage`에 키를 강제로 저장하지 않고 화면만 가린 채 안전하게 진입 처리(`hideStartScreen()`)합니다.

---

## 검증 계획

### 수동 검증
1. 브라우저에서 `/api/get-env-key` 주소에 직접 접속했을 때 `{"exists": true}` 또는 `{"exists": false}` 형태로 노출되며 실제 키값은 전혀 드러나지 않는지 확인.
2. 로컬 스토리지에 키가 비어있는 상태에서 서버 키 유무에 따른 로그인 화면 자동 패스(Skip) 기능이 예전과 완전히 동일하게 작동하는지 검증.
