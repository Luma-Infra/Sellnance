# 데이터 정렬 비어있는 값(하이픈/-) 항상 최하단 배치 계획

모든 컬럼의 정렬 시, 화면상에 값이 없어서 하이픈(`-`)으로 표시되는 항목(실제 값이 null, undefined이거나 거래량/시총 등이 0인 경우)을 오름차순/내림차순 상관없이 항상 최하단에 강제 배치하도록 정렬 비교 알고리즘을 개선합니다.

## 제안된 변경 사항

### 정렬 로직 수정

#### [MODIFY] [table_sort.js](file:///c:/Users/78831/Sellnance/static/table_sort.js)
- `simpleSortData` 함수 내의 Schwartzian Transform 단계에서 각 컬럼의 특성에 따라 값이 없는 상태(`isEmpty`)를 정밀하게 정의합니다.
  - **VolumeUpbit (U-Vol)**: 값이 `0`이거나 표시 텍스트가 `"-"`인 경우
  - **Volume (Volume)**: 값이 `0`이거나 표시 텍스트가 `"-"`인 경우
  - **MarketCap (Mcap)**: 값이 `0`이거나 표시 텍스트가 `"-"`인 경우
  - **VMC**: 값이 `0`이거나 표시 텍스트가 `"-"`인 경우
  - **Funding (Funding)**: 값이 `0`이거나 표시 텍스트가 `"-"`인 경우
  - **Kimchi (김프)**: 김프 매칭 라벨(`Kimchi_Label`)이 `"-"`이거나 비어있는 경우
- `isEmpty = true`로 판정된 데이터는 오름차순/내림차순(isAsc)에 관계없이 정렬 결과에서 항상 배열의 가장 마지막(최하단)으로 정렬되도록 비교 연산을 유지합니다.

## 검증 계획

### 수동 검증
1. 브라우저에서 **U-Vol(24h)** 헤더를 클릭하여 내림차순(▼) 및 오름차순(▲)으로 각각 정렬해 봅니다.
2. 두 정렬 상태 모두에서 업비트에 상장되지 않아 값이 없는 코인들(하이픈 `-` 표시)이 항상 리스트의 최하단에 배치되는지 확인합니다.
3. **김프**, **VMC**, **시총(Mcap)**, **펀비(FUNDING)** 등 다른 정렬 컬럼들도 동일하게 값 없는 항목이 최하단에 고정되는지 검증합니다.
4. `npx vite build`가 오류 없이 성공하는지 테스트합니다.
