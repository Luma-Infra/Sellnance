# 구현 계획서 - 테이블 실시간 갱신 중단 및 특정 Row 겹침/비어버림 UI 버그 수정

## Goal Description
다음 두 가지 치명적인 테이블 UI 버그를 해결합니다.

1. **코인 클릭 시 실시간 업데이트 및 정렬(경주마) 중단 오류:**
   - **원인:** 특정 코인을 클릭(선택)할 때 `ui_control.js` 내 `selectSymbol`이 전역 마켓 모드인 `store.currentMarket`을 해당 코인의 거래소 마켓(예: `"UPBIT"`)으로 강제 변경합니다. 이 때문에 테이블 렌더링 및 틱 필터링 로직들이 선물 모드(`FUTURES`)를 이탈하여 바이낸스 선물 틱의 갱신이 원천 차단됩니다.
   - **해결책:** **"테이블 뷰 기준의 활성 마켓"**(`store.currentMarket`)과 **"차트/호가창 등 상세 뷰 기준의 활성 마켓"**(`store.currentChartMarket`)을 분리하여 개별 코인 선택이 테이블 정렬 및 갱신 주기에 영향을 주지 않도록 격리합니다.

2. **31등에 코인이 쌓이고 33등, 36등 등 특정 Row가 비어버리는 오류:**
   - **원인:** `table_render.js`의 `renderTable(isRealtime)` 내에서 실시간 갱신 시 위치 재배치 조건식인 `needsPositionUpdate`가 30등 밖의 코인들에 대해서는 업데이트를 생략하도록 막고 있습니다.
     ```javascript
     const needsPositionUpdate = !isRealtime || i < 30 || oldIndex < 30 || isNaN(oldIndex);
     ```
     이 때문에 30등 밖에서 순위 변동이 일어난 코인들은 실제 데이터 인덱스는 바뀌었으나 DOM 위치(`translateY`)가 제자리로 갱신되지 못해 특정 위치에 겹쳐서 쌓이고, 이동했어야 할 자리는 비어 있게 됩니다.
   - **해결책:** 실시간 정렬 여부와 관계없이 순위 인덱스가 달라졌다면(`oldIndex !== i`) 무조건 위치 재배치와 등수 갱신을 하도록 방어 조건을 추가합니다.

## Verification Plan

### Manual Verification
1. 바이낸스 선물 탭(예: F-FUT)을 선택하여 선물 코인들이 1초 주기로 실시간 업데이트 및 정렬(경주마)이 활발하게 일어나는지 확인합니다.
2. 이 상태에서 **바이낸스 미상장 업비트 전용 코인**의 Row를 클릭합니다.
   - 우측 영역에 해당 코인의 차트 및 호가창이 업비트 기준으로 정상 갱신되어 움직이는지 확인합니다.
   - 클릭 이후에도 좌측 테이블 목록의 선물 코인들이 갱신을 멈추지 않고 1초 주기로 실시간 정렬(경주마)이 계속 구동되는지 검증합니다.
3. 스크롤을 천천히 내리거나 실시간 정렬이 발생할 때, 31등 이하의 등수(예: 31등, 33등, 36등)에서 코인이 겹쳐서 표시되거나 행이 비어버리는 현상이 완전히 소진되었는지 눈으로 직접 검증합니다.
