# 성능 디버거 렉 격리 및 aggTrade 주기 조절 추가 완료 보고

대시보드 실시간 갱신 시 발생하는 프레임 드랍 요인들을 정밀하게 대조 테스트할 수 있도록 디버거의 내부 연산 통제 구조를 **상호 독립적인 분기**로 완벽하게 리팩토링하고, **aggTrade(체결 틱)의 실시간 갱신 주기(Raw, 100ms, 500ms, 1500ms) 조절 기능**을 추가 완료했습니다.

## 1. 렉 유발 요인 격리 리팩토링 및 aggTrade 주기 조절 추가

각 스위치와 주기 조절 기능이 제어하는 연산 범위를 논리적/물리적으로 완전히 분할하여 중복 제어 및 연산 누수를 영구적으로 해결했습니다.

```mermaid

        I[blockKimchi] -->|독립 제어| J[차트 김프 및 테이블 실시간 김프 연산 자체 스킵]
```

### A. aggTrade 주기 조절 기능 구현 상세
* **기능 소개**: 바이낸스 및 업비트 실시간 체결 틱 데이터를 수신할 때 DOM 갱신과 Flash 애니메이션이 과부하를 주지 않도록 **쓰로틀링(Throttling) 주기**를 4단계(Raw, 100ms, 500ms, 1500ms)로 조절할 수 있습니다.
* **메모리 보존**: 쓰로틀링 중에도 메모리 상의 시세 데이터(`row.Price_Raw` 등)는 계속 최신으로 업데이트되어 데이터 정합성을 유지하며, DOM 렌더링 큐 등록만 지정된 주기만큼 지연/스킵됩니다.
* **수정 파일**:
  * [_store.js](file:///c:/Users/kmj/Sellnance/static/_store.js): 전역 스토어에 `aggTradeInterval: 0` (0=Raw) 변수 추가.
  * [index.html](file:///c:/Users/kmj/Sellnance/templates/index.html): 우측 하단 렉 디버거 패널 최하단에 HSL 스타일로 디자인된 Raw, 100ms, 500ms, 1500ms 4단 그리드 버튼 컴포넌트 추가.
  * [ui_control.js](file:///c:/Users/kmj/Sellnance/static/ui_control.js): `setAggTradeInterval` 전역 함수를 추가하여 클릭 시 스토어 값을 바꾸고 활성화된 버튼에 Accent 스타일링 하이라이팅 처리.
  * [stream.js](file:///c:/Users/kmj/Sellnance/static/stream.js): 실시간 틱 데이터가 들어올 때 각 Ticker의 `lastRenderMap` 타임스탬프를 대조하여 지정 주기가 경과하지 않았으면 DOM 쓰기 프로세스를 조기 반환(Early Return) 처리.
