# 성능 최적화 및 두 부모(좌/우 DOM 차단) 기능 극대화 기획서

본 보고서는 마우스, 메모리, DOM 브라우저 연산 오버헤드를 극적으로 제거하고 사용자가 원하는 최적의 UX를 달성하기 위해 **기존 로직 훼손 없이(코드 삭제 불허, 주석 처리 및 조건부 얼리 리턴 활용)** 두 부모 토글(`blockRightDom`, `blockLeftDom`) 상태 하에서 모든 무거운 렌더링/CSS 연산을 완벽하게 침묵시키는 최적화 계획안을 담고 있습니다.

---

## 1. 핵심 최적화 대상 및 대응 방안

### ① `aggTrade` 웹소켓 입구 레벨 500ms 쓰로틀링 강제 주입
- **현황**: 바이낸스, 업비트 등 실시간 체결(aggTrade) 정보가 초당 수십~수백 건 폭발적으로 유입되어 JS 연산 큐 및 가비지(GC) 메모리 적체가 발생합니다.
- **개선안**: 웹소켓 메시지가 수신되어 파싱되는 입구(`stream_table.js` 등)에서 `aggTrade` 이벤트에 한해 이전 처리 시간 기준 **최소 500ms 단위 윈도우 스냅 쓰로틀**을 강제로 겁니다. 500ms 미만 간격으로 도달하는 체결 틱은 연산 버퍼조차 태우지 않고 즉시 리턴시킵니다.

### ② 좌측 테이블 DOM 차단 시 실시간 순위 재배치 침묵
- **현황**: `table.js`의 1초 주기 순위 재배치 루프(`setInterval`)가 `blockLeftDom` 또는 `blockSort`가 켜진 상태에서도 백그라운드에서 매초 `applyRealtimeSort()`를 호출하여 불필요한 레이아웃 연산을 시도합니다.
- **개선안**: 1초 주기 타이머 초입에 `if (store.blockLeftDom || store.blockSort) return;` 방어 조건을 주입하여 무거운 정렬 및 FLIP 레이아웃 애니메이션 연산을 완전히 무력화시킵니다.

### ③ 가격 변동 플래시(CSS Flash) 연산 차단
- **현황**: 시세 변동 시 셀이 초록색/빨간색으로 번쩍이는 효과(`applyPriceFlash`)가 빈번히 일어나며 브라우저 Paint/GPU 가속 오버헤드를 발생시킵니다.
- **개선안**: `applyPriceFlash` 함수의 입구에 `if (store.blockLeftDom || store.blockTableUpdate) return;` 을 추가하여, 좌측 테이블이 차단되었거나 테이블 업데이트가 꺼졌을 때는 CSS 추가/삭제 및 프레임 큐 요청을 즉각 바이패스합니다.

## 3. 검증 계획

- 성능 디버거에서 `좌측 테이블 DOM 차단` 및 `우측 패널 DOM 차단`을 켰을 때 CPU 사용량이 0%대에 수렴하고 렌더링 프레임(FPS)이 60으로 완벽히 고정되는지 확인.
- 소켓 유입 로그가 500ms 주기로 디바운스 및 쓰로틀링되어 입구 레벨에서 통제되는지 정밀 모니터링.

## 4. Go & SolidJS 마이그레이션 기술 추론 및 대비 전략

### 🐹 Go 백엔드 마이그레이션 관점
- **시세 게이트웨이 병목의 서버 측 격리**: 현재 클라이언트 브라우저 입구에서 JS 스레드로 처리하는 `500ms 쓰로틀링` 필터는 Go 백엔드 마이그레이션 시 **서버 단의 고성능 채널(Go Channel) 및 타이머(Ticker)**를 통해 1차적으로 일괄 정제하여 클라이언트로 쏠 수 있습니다.
- **Goroutine 병렬 처리 활용**: 수백 개의 웹소켓 커넥션에서 인입되는 원시 체결 틱을 백엔드 고루틴 풀에서 메모리 공유 없이 스냅샷 캐싱 처리하므로, 프론트엔드가 감당해야 할 데이터 스루풋 파싱 연산량을 물리적으로 서버 측으로 90% 이상 전가시킬 수 있어 프론트엔드의 부하 분산에 매우 효과적입니다.

### ⚡ SolidJS 프론트엔드 마이그레이션 관점
- **미세 반응형(Fine-grained Reactivity) 설계 적용**: Vanilla JS의 전형적인 DOM 직접 타격 방식(`getElementById` + `textContent` 갱신 큐) 대신, SolidJS의 `createSignal` 또는 `createStore`를 활용하여 가격 셀을 단독 시그널 노드로 래핑합니다.
- **구독(Subscription) 제어의 직관성**: `store.blockTableUpdate` 혹은 `store.blockLeftDom` 상태를 하나의 Signal로 선언하면, 리액티브 바인딩을 통해 데이터가 유입되더라도 해당 돔 엘리먼트 노드의 반응(Effect)만을 완전히 **일시중단(Suspend)/재개(Resume)**시킬 수 있습니다.
- **FLIP 및 정렬 렌더링 오버헤드 완화**: 가상 DOM Diffing 계산서가 없는 SolidJS의 특성상 컴포넌트 전체가 재생성되지 않고 정렬된 포인터 DOM만 컴파일된 네이티브 돔 위치에 꽂히기 때문에, 애니메이션 가속 부하가 최소화되며 500ms 소켓 쓰로틀 게이트와 결합 시 극강의 성능 향상을 기대할 수 있습니다.
