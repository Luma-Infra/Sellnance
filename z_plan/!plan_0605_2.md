# 주식 토큰 즐겨찾기(Favorites) 토글 버그 해결 보고서

SAMSUNG, HYUNDAI, SKHYNIX 등 주식/전통자산 파생 토큰들의 즐겨찾기(별 아이콘) 기능이 작동하지 않던 버그를 분석하고, 하드코딩 없이 100% 데이터 기반으로 해결한 내역을 정리합니다.

## 🐛 버그 원인 분석

1. **UID 중복 문제 (1차 원인)**:
   - 코인마켓캡(CMC)에 등록되지 않은 주식 파생 토큰들은 백엔드 조립 시 `final_ucid`가 빈 문자열(`""`)로 획득되었습니다.
   - 이로 인해 프론트엔드가 모든 주식 자산을 동일한 `""` 식별자로 판정하여, 로컬 스토리지(`sellnance_favs`) 내에 중복 적재 및 오작동이 일어났습니다.
   - *(이전 단계에서 UID가 없을 때 본래의 `base` 심볼명인 `"SAMSUNG"`, `"HYUNDAI"` 등으로 대체되도록 1차 조치는 완료된 상태였습니다.)*

2. **주식 자산 오분류 및 필터링 오작동 (근본 원인)**:
   - 프론트엔드 [isStockCoin()](file:///c:/Users/78831/Sellnance/static/table_filter.js#L6-L11) 함수는 자산의 영문명(Name)에 `"stock"` 또는 `"derivative"`가 포함되어 있는지를 기준으로 주식 자산을 판단했습니다.
   - 하지만 한국 주식 토큰들은 CMC 조회가 실패하여 백엔드에서 이름이 원본 심볼명인 `"SAMSUNG"`, `"HYUNDAI"`, `"SKHYNIX"`로 그대로 세팅되어 들어왔습니다.
   - 이에 따라 주식 토큰임에도 불구하고 **일반 암호화폐로 오분류**되었습니다.
   - 결과적으로 주식 탭(**B-STOCK** 필터 활성화 상태)에서 해당 토큰들이 리스트에서 아예 제외(필터링)되어 노출되지 않았고, 이로 인해 즐겨찾기 추가/삭제 동작 시 필터와 충돌하며 즐겨찾기 탭(`FAV`) 전환 시 정상 노출되지 않았습니다.

## 🛠️ 해결 방식 (100% 데이터 기반 리버스 트래킹)

하드코딩 화이트리스트 없이 바이낸스 엔진 자체 데이터로부터 주식 토큰을 정확히 판별할 수 있도록 수정했습니다.

### 1. 백엔드 바이낸스 선물 메타데이터 추출 (`modules/exchange_api.py`)
- 바이낸스 선물 `exchangeInfo` API에서 제공하는 고유 스펙 정보를 추출하는 파이프라인을 구축했습니다:
  - **`underlyingType`**: 주식 및 전통 자산 계열은 `"EQUITY"` 또는 `"KR_EQUITY"` 값을 가집니다.
  - **`contractType`**: 주식 토큰은 `"TRADIFI_PERPETUAL"` 스펙을 갖습니다.
- 수집된 세부 스펙을 바탕으로 `binance_data[ticker]`에 `"underlying_type"` 및 `"contract_type"` 값을 바인딩하여 데이터 조립기로 넘겨줍니다.

### 2. 최종 장부(Row)에 `Is_Stock` 필드 주입 (`modules/builder_binance.py`)
- 바이낸스 데이터 빌더가 최종 조립 시 아래의 규칙으로 `Is_Stock` 여부를 판정해 JSON 객체에 추가합니다:
  ```python
  u_type = str(b_info.get("underlying_type", "")) if isinstance(b_info, dict) else ""
  c_type = str(b_info.get("contract_type", "")) if isinstance(b_info, dict) else ""
  is_stock = ("EQUITY" in u_type) or (c_type == "TRADIFI_PERPETUAL")
  ```
- 이를 통해 `SAMSUNG`, `HYUNDAI`, `SKHYNIX` 뿐만 아니라 향후 추가되는 모든 전통자산(AAPL, TSLA 등)도 **자동으로 `Is_Stock: true`로 분류**됩니다.

### 3. 프론트엔드 공통 판별기 업데이트 (`static/table_filter.js`)
- [isStockCoin()](file:///c:/Users/78831/Sellnance/static/table_filter.js#L6-L12) 함수가 백엔드에서 전달받은 `Is_Stock === true` 속성을 최우선으로 검사하도록 개선했습니다:
  ```javascript
  export function isStockCoin(row) {
    if (!row) return false;
    if (row.Is_Stock === true) return true; // 백엔드 바이낸스 분류값 최우선 반영!
    const name = (row.Name || "").toLowerCase();
    if (name.includes("rootstock")) return false;
    return name.includes("stock") || name.includes("derivative");
  }
  ```

## 🧪 검증 결과

- 백엔드 캐시 재생성 후 `/api/market-data` API 응답 데이터 검증 완료:
  - `TSLA`, `AAPL` (해외 주식): `Is_Stock: true` (정상)
  - `SAMSUNG`, `HYUNDAI`, `SKHYNIX` (국내 주식): `Is_Stock: true` (정상)
  - `BTC`, `ETH` (일반 암호화폐): `Is_Stock: false` (정상)
- 이를 통해 **B-STOCK** 필터가 작동할 때 한국 주식 토큰들도 완벽히 식별되어 화면에 보이고, 즐겨찾기(`★`)를 클릭하면 노란별/파란별로 올바르게 토글되며 **FAV/FAV2** 즐겨찾기 탭에서도 정상 작동합니다.
