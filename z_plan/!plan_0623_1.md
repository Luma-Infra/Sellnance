# 구현 계획서 - 중복 제거, 라우팅 수정 및 실시간 차트 범례 추적 개선

다음 핵심 문제를 해결합니다:
1. **사일런트 동기화 시 중복 제거**: 클라이언트가 백그라운드 데이터를 동기화할 때 `TAIKOUSDT`와 `TAIKOKRW`처럼 동일 코인이 중복 행으로 표시되는 문제를 수정합니다.
2. **주소창 해시 라우팅**: 활성화된 마켓 선택에 따라 URL 해시를 동적으로 업데이트하고(`#TAIKOKRW` (업비트/빗썸 KRW), `#TAIKOUSDT` (바이낸스/바이비트 USDT)), 새로고침이나 직접 접속 시에도 접미사(Suffix)를 분석하여 해당 코인과 마켓을 올바르게 매칭합니다.
3. **오전 9시 KST 리셋 및 실시간 동기화**: 백엔드에서 빗썸의 시가(`utc0_open`)를 정상적으로 수집하도록 개선하고, 프론트엔드에서 `utc0_open_KRW`가 없는 바이낸스 상장 코인을 국내 마켓으로 볼 때도 달러 시가와 환율을 활용해 `Change_Today_Raw`가 실시간으로 올바르게 변동되도록 보완합니다.
4. **차트 범례(Legend) 마우스 추적 오류 수정**: 캔들 영역을 벗어나 우측 빈 공간(미래 영역)에 마우스가 위치할 때 `displayName`에 접미사(예: `KRW`)가 붙어 `rowInfo`를 찾지 못하고 TypeError 크래시가 발생하던 문제를 해결하여, 범례에 마지막 캔들의 OHLC 값이 항상 정상 노출되도록 수정합니다.

## 검증 계획

### 자동 빌드 테스트
- 변경 내용 반영 후 `npx vite build`를 실행하여 컴파일 오류가 없는지 검증합니다.

### 수동 검증 테스트
1. 브라우저로 `http://127.0.0.1:8000/#TAIKOKRW`에 직접 접속합니다.
   - 업비트 TAIKO 캔들이 정상적으로 로드되는지 확인합니다.
   - 우측 상단 헤더에 "TAIKO (Taiko)"가 정상 출력되는지 확인합니다.
   - 거래소 배지에서 "UPBIT"가 활성화되어 있고 주소창의 해시가 `#TAIKOKRW`로 정상 유지되는지 확인합니다.
2. 마켓 배지에서 "B-FUT"를 클릭합니다.
   - 바이낸스 선물 TAIKO 차트로 정상 전환되는지 확인합니다.
   - 주소창의 해시가 `#TAIKOUSDT`로 즉시 업데이트되는지 확인합니다.
3. 백그라운드 1분 동기화(Silent Sync)가 실행된 후, 테이블 상에 `TAIKOKRW`와 `TAIKOUSDT`가 중복해서 생겨나지 않는지 모니터링합니다.
4. 마우스 포인터를 차트 캔들 우측의 미래 빈 영역으로 옮깁니다. 바이낸스 코인과 업비트 전용 코인(예: UP2) 모두 크로스헤어가 캔들을 벗어나도 범례(Legend) 영역에 마지막 캔들의 OHLC 값이 올바르게 고정되어 노출되는지 확인합니다.
