# 하이브리드 최적화 모델 (정적 식별 정보 + 동적 지표 데이터 분리) 도입 계획

기존의 가상화 스크롤 방식은 화면 밖에 위치한 행들에 대해 완전히 비어있는 껍데기(`EMPTY_ROW_HTML`)를 노출했습니다. 이로 인해 빠른 스크롤 시 화면에 하얀 공백(Placeholder)이 많이 보여 UX 완성도가 떨어지는 아쉬움이 있었습니다.

이 계획은 변하지 않는 **정적 식별 정보**(순위 번호, 즐겨찾기 별 버튼, 코인 로고, 티커명, 코인 이름)는 최초 DOM 풀 생성 시 800개 전체 행에 대해 즉시 렌더링하고, 무거운 CPU 연산 및 DOM 노드가 많이 필요한 **동적 지표 데이터**(가격, 볼륨/시총, 김프/펀딩비, 상장 거래소 그리드, 상장일)만 가시 영역(또는 상위 30개)에 들어왔을 때 지연 로딩(Lazy Loading) 및 업데이트하는 **하이브리드 부분 지연 로딩(Hybrid/Partial Lazy Loading)** 아키텍처를 도입하기 위한 것입니다.

## Proposed Changes

### 1. 정적/동적 레이어 렌더링 함수 분할 및 리팩토링
#### [MODIFY] [table_render.js](file:///c:/Users/78831/Sellnance/static/table_render.js)
- **`updateRowStaticHTML(tr, row)` 신설**:
  - `col-asset` 내부(순위, 별 버튼, 로고, 티커/이름)를 렌더링하고, 나머지 수치 지표 영역(`col-price`, `col-vol-b`, `col-vol-u`, `col-kimch`, `col-exch`, `col-listing`)은 `-` 혹은 비어있는 Placeholder 상태의 테이블 셀(`td`) 구조로 생성합니다.
  - 브라우저 레이아웃 부하가 적어 최초 풀 생성 시 800개 전체에 무리 없이 적용 가능합니다.
  - `tr.dataset.metricsRendered = "false"` 상태 속성을 부여합니다.
- **`updateRowDynamicHTML(tr, row)` 신설**:
  - 가격 정보(`col-price` 하위 구조), 바이낸스 볼륨/시총, 업비트 볼륨/VMC, 김프/펀딩비, 8대 거래소 필터링 아이콘 그리드, 상장일을 렌더링합니다.
  - 렌더링 완료 후 `tr.dataset.metricsRendered = "true"` 속성을 지정하고, 즉시 `window.updateRowPriceDisplay(tr, row)`를 호출하여 가격 변동을 실시간 싱크합니다.
- **`updateRowInnerHTML(tr, row)` 래퍼 함수로 변경**:
  - 기존 코드 및 외부 모듈(`chart_data.js` 등)과의 호환성을 100% 유지하기 위해, 이 함수는 `updateRowStaticHTML(tr, row)`와 `updateRowDynamicHTML(tr, row)`를 순차적으로 호출하여 동작하게 수정합니다.
- **`EMPTY_ROW_HTML` 제거**:
  - 더 이상 아무 내용 없는 공백 껍데기를 그리지 않으므로 해당 상수를 삭제합니다.

### 2. 가상화 테이블 렌더러 (`renderTable`) 리팩토링
#### [MODIFY] [table_render.js](file:///c:/Users/78831/Sellnance/static/table_render.js)
- **최초 DOM 풀 생성 단계**:
  - 800개 전체 행에 대해 `updateRowStaticHTML`을 호출하여 기본 뼈대와 코인 명칭을 렌더링합니다.
  - 상위 30개 행에 대해서만 `updateRowDynamicHTML`을 즉시 호출하여 지표를 렌더링하고 `visibleSymbols`에 등록합니다.
- **수동 정렬 및 필터링 (`!isRealtime`) 단계**:
  - 800개 전체의 순서를 맞추어 배치하되, 가시 영역이나 상위 30개 행에 해당하는 경우에만 동적 데이터(`updateRowDynamicHTML`)를 조건부 렌더링합니다.
- **실시간 갱신 및 순서 변동 단계 (`isRealtime`)**:
  - 정렬 순서에 변동이 생기거나, 혹은 가시 영역 내부를 갱신할 때 `updateRowDynamicHTML`이 호출되지 않았거나 설정(화폐 모드, 언어 등)이 바뀐 경우에만 선별적으로 렌더링을 유도합니다.
- **`IntersectionObserver` 콜백 단계**:
  - 가시 영역 내부로 스크롤되어 들어올 때, 해당 행의 `metricsRendered === "false"`인 경우 `updateRowDynamicHTML`을 호출하여 미처 로딩되지 않은 실시간 지표 데이터를 채워 넣습니다.

## Verification Plan

### Manual Verification
1. **스크롤 공백 검사**:
   - 스크롤을 아주 빠르게 내리더라도 흰색 빈 상자 대신 코인의 로고, 티커, 한글/영문 이름이 고정되어 정적으로 노출되는지 확인합니다.
   - 스크롤이 멈추거나 300px 이내 영역으로 유입될 때 실시간 지표(가격, 볼륨, 김프, 거래소 아이콘 그리드)가 부드럽고 신속하게 채워지는지 확인합니다.
2. **성능 벤치마크**:
   - 800개 코인 정렬 시 및 실시간 웹소켓 가격 변동 주기 시 브라우저 프레임 드랍(Stuttering) 현상 없이 렌더링이 부드러운지 검증합니다.
3. **호환성 검증**:
   - `chart_data.js` 등 외부 모듈에서 호출하는 `updateRowInnerHTML`이 오동작 없이 정상 작동하는지 확인합니다.
