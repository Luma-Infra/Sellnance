# 사용자 로컬 환경보존 UX 설계 보고서

본 보고서는 **Sellnance** 서비스의 사용자 설정(테마, UI 접기/폴딩 상태 등)을 서버 비용 0원을 달성하면서 브라우저 환경에서 가장 이상적으로 보존하기 위한 저장소 매핑 및 UX 시나리오를 정의합니다.

---

## 1. 보존 대상 항목 및 스토리지 매핑 제안

사용자의 행동 및 상태를 **LocalStorage**, **SessionStorage**, **Cookie**의 특성에 맞춰 다음과 같이 분류하여 보존합니다.

| 저장 항목 | 저장 데이터 스펙 | 추천 스토리지 | 의사결정 및 UX 근거 |
| :--- | :--- | :--- | :--- |
| **테마 설정 (Theme)** | `"binance"` (다크) / `"upbit"` (라이트) | **LocalStorage** | 새로고침이나 브라우저를 닫았다 열었을 때 눈부심(Flash)을 방지하기 위해 영구 보존이 필수입니다. |
| **사이드바 접기 (Sidebar Collapse)** | `true` (접힘) / `false` (펼침) | **LocalStorage** | 모니터 크기가 작은 쌀먹 유저가 화면 영역을 넓게 쓰는 상태를 브라우저를 재부팅해도 그대로 기억해야 합니다. |
| **차트 헤더 접기 (Header Fold)** | `true` (접힘) / `false` (펼침) | **LocalStorage** | 차트 영역을 세로로 널찍하게 쓰는 고유 UX 상태를 영구 기억하는 것이 자연스럽습니다. |
| **단기 퀵 서칭 입력 버퍼** | 최근 검색 텍스트 등 | **SessionStorage** | 탭을 닫으면 사라지는 단기성 작업 데이터이므로 세션이 끝날 때 휘발되는 것이 UX상 깔끔합니다. |
| **서버 통신 토큰 / API 키** | `CMC_API_KEY` | **LocalStorage** | 매번 입력할 수 없는 민감 정보이므로 영구 보존합니다. |

---

## 2. 기술별 한계 및 UX 보안 전략 (0원 쌀먹용)

1. **LocalStorage (영구 보존)**
   * **용도**: 테마, 즐겨찾기, 폴딩 상태, 영구 API 키
   * **특징**: 만료 기간이 없어 사용자가 직접 삭제하지 않는 한 평생 보존됩니다.
   * **주의**: 쿠키 정리 시 지워질 위험이 있습니다.

2. **IndexedDB (대피소)**
   * **용도**: 로컬 스토리지 한계(5MB)를 초과하는 대용량 데이터 (예: 사용자가 잔뜩 그려놓은 차트 작도 드로잉 툴 데이터)
   * **특징**: 쿠키 단순 정리로 쉽게 날아가지 않는 강력한 격리 저장 영역입니다.

3. **Supabase Direct (최종 보존막 - 0원)**
   * **동작**: 사용자가 '구글 연동' 또는 '익명 기기 바인딩'을 활성화할 경우, 클라이언트 브라우저가 직접 Supabase DB로 설정 정보를 1초 주기로 자동 저장(Auto-Save) 및 불러오기(Fetch)를 수행합니다.

---

## 3. 향후 구현 방향

* **테마 설정 로컬 보존**: 페이지 최초 로딩 시 `localStorage.getItem("sellnance_theme")`을 조회하여 바디 테마 클래스(`theme-binance` 또는 `theme-upbit`)를 즉시 로드합니다.
* **폴딩 상태 로컬 보존**: 사이드바 및 헤더 폴딩 전환 이벤트 발생 시 `localStorage.setItem("sellnance_sidebar_collapsed", ...)` 형태로 상태를 기억하고 돔 로드 시 적용합니다.
