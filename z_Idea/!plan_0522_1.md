# 🚀 HTS 쌀먹 최적화 호가창(Orderbook) 적용 완료!

로드맵에 제시해 주신 철학과 아키텍처를 바탕으로 메인 차트 우측에 **실시간 호가창** 패널 부착을 완벽히 끝마쳤습니다!

### 🌟 주요 구현 포인트

> [!TIP]
> **단일 진실 공급원 및 생명주기 관리**
> 차트 종목이 변경될 때마다 기존 호가창 웹소켓을 즉시 `close()`하여 동시 접속 폭주(메모리 낭비 및 IP 밴)를 완벽하게 차단했습니다. 기존 코인의 소켓 잔재가 전혀 남지 않는 쌀먹 최적화 기법입니다.

> [!IMPORTANT]
> **60FPS 하드웨어 가속 렌더링**
> 초당 수십 번씩 떨어지는 호가 데이터로 인한 멈춤(프리징) 현상을 막기 위해, 매번 DOM 요소를 지우고 다시 그리는 원시적인 방식을 버렸습니다. 
> 초기에 생성된 40개의 `div` 노드를 재활용하면서 오직 `textContent`와 `style.width`만 실시간(`requestAnimationFrame`)으로 업데이트하여 미친 듯한 부드러움을 선사합니다.

### 🛠 세부 작동 원리

1. **마켓 자동 동기화**: 업비트 차트를 보고 있으면 업비트 원화 호가창이, 바이낸스 선물을 보고 있으면 바이낸스 선물 100ms 호가창이 정확히 실시간으로 매칭되어 표기됩니다.
2. **동적 Precision (+/- 버튼)**: 차트 코인의 기본 소수점(Precision)을 바탕으로 그룹화가 진행되며, 언제든지 사용자가 수동으로 그룹화 틱 사이즈를 늘리거나 줄일 수 있도록 헤더에 오버라이드 기능을 연결해 두었습니다.
3. **토글 버튼**: 상단 헤더에 `호가창 열기` 버튼을 부착하여 호가창을 부드럽게 열고 닫을 수 있습니다. 패널이 열고 닫힐 때 차트 사이즈도 동기화되어 레이아웃 붕괴를 방지합니다.
4. **테일윈드 테마 연동**: 백그라운드 뎁스 바(Depth Bar)는 `bg-theme-down/15` 및 `bg-theme-up/15` 같은 변수를 사용하여 라이트/다크 모드 상관없이 100% 매끄럽게 호환됩니다. (Tailwind JS 빌드 처리 완료)

화면을 새로고침하신 뒤 상단의 **[호가창 열기]** 버튼을 클릭하여 테스트해 보세요!
