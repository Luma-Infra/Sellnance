# 구현 계획서: 차트 좌측 그리기 툴바 기능 추가

본 계획서는 차트 영역 좌측에 트레이딩뷰 스타일의 플로팅 그리기 툴바를 배치하고, 추세선/가로선/측정 자 도구 및 삭제 기능을 추가하기 위한 명세입니다.

## 사용자 검토 필요 사항

> [!IMPORTANT]
> - 그리기 데이터의 지속적인 메모리 관리를 위해 `store.drawings` 전역 상태 객체를 생성합니다.
> - 차트 줌 및 스크롤 시 그린 선들이 캔버스 좌표에 맞춰 정확히 고정되어 연동되도록 `lightweight-charts` 커스텀 렌더링 프리미티브를 구현합니다.
> - 차트 좌측에 다음과 같은 **5가지 아이콘의 플로팅 툴바**를 배치합니다:
>   1. **기본 커서 (Cursor)**: 그리기 모드를 끄고 일반 차트 탐색(줌/스크롤) 상태로 복원
>   2. **추세선 (Trend Line)**: 시작점 클릭 후 드래그/클릭하여 대각선 그리기
>   3. **가로선 (Horizontal Line)**: 클릭한 가격 위치에 가로 지지/저항선 생성
>   4. **측정 자 (Ruler)**: 기존의 등락률/봉 개수 측정 도구 활성화
>   5. **전체 지우기 (Trash)**: 차트 위의 모든 선과 측정값 일괄 삭제

## 제안된 변경 사항

### 1. 전역 상태 저장소 확장

#### [MODIFY] [static/_store.js](file:///c:/Users/kmj/Sellnance/static/_store.js)
`store` 객체에 다음 변수들을 추가합니다:
- `store.activeTool`: `'cursor' | 'trendline' | 'horizontal' | 'measure'` (기본값: `'cursor'`)
- `store.drawings`: `{ trendlines: [], horizontals: [] }` (그려진 선 정보들 저장)
- `store.drawingStart`: `{ logical, price }` (추세선 그리기 시작 좌표 임시 저장)
- `store.drawingTempEnd`: `{ logical, price }` (추세선 드래그 중인 끝 좌표 임시 저장)
- `store._drawingPrimitive`: 커스텀 그리기 프리미티브 인스턴스 보관용

---

### 2. UI 툴바 및 스타일 추가

#### [MODIFY] [templates/index.html](file:///c:/Users/kmj/Sellnance/templates/index.html)
차트 컨테이너 `#chart-wrapper` 내부의 좌측에 절대 위치(`absolute`)로 정렬된 수직형 플로팅 툴바를 삽입합니다:
- 5개의 도구 버튼 (커서, 추세선, 가로선, 자, 휴지통) 배치.
- 테마별로 조화로운 프리미엄 글래스모피즘(반투명/테두리 효과) 디자인 적용.
- 현재 선택된 도구 버튼에 테마 강조 하이라이트 클래스 적용.

### 3. 그리기 렌더러 및 이벤트 처리 로직

#### [NEW] [static/chart_draw.js](file:///c:/Users/kmj/Sellnance/static/chart_draw.js)
새로운 스크립트 파일을 생성하여 다음 로직을 구현합니다:
- `DrawingPrimitive` 클래스 정의 (Lightweight Charts `Attached` / `Detached` / `PaneViews` API 호환 구현).
- `DrawingPaneView` 및 `DrawingPaneRenderer` 클래스를 구현하여 캔버스 위에 그려진 모든 추세선, 가로선 및 그리는 도중의 임시 선을 렌더링.
- 차트 마우스 이벤트를 바인딩하는 `initDrawingEvents(container)` 구현:
  - **추세선 (`trendline`)**:
    - 첫 번째 클릭: `store.drawingStart` 좌표 지정 및 그리기 상태 활성화.
    - 이동: `store.drawingTempEnd` 끝 좌표 실시간 갱신 및 렌더러 업데이트.
    - 두 번째 클릭: 최종 시작점-끝점 값을 `store.drawings.trendlines`에 추가하고 임시 상태 초기화.
  - **가로선 (`horizontal`)**:
    - 클릭: 클릭한 위치의 가격 값을 `store.drawings.horizontals`에 추가.
  - **측정 자 (`measure`)**:
    - 기존 `chart_measure.js` 자 기능 연동 활성화.
- 전역 헬퍼 `selectDrawingTool(toolName)`을 정의하여 HTML 툴바 클릭 시 그리기 모드 전환.

## 검증 계획

### 수동 검증
- 웹 애플리케이션 접속 후 차트 좌측 툴바 생성 확인.
- **추세선**: 도구 클릭 후 차트 위에 대각선을 그려보고, 차트를 스크롤하거나 줌 조절을 해도 대당 캔들에 선이 완벽히 고정되어 움직이는지 검증.
- **가로선**: 도구 클릭 후 특정 가격 클릭 시 가로선이 그려지는지 검증.
- **측정 자**: 도구 클릭 후 드래그 시 기존 측정 툴이 자연스럽게 작동하는지 검증.
- **기본 커서**: 일반 커서 선택 시 차트 그리기가 중단되고 일반 스크롤/줌 조절 기능만 작동하는지 확인.
- **휴지통**: 클릭 시 차트 상의 모든 선들이 깨끗하게 지워지는지 확인.
