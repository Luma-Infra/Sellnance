# 이행 계획서 - 별 모양 통일 및 5초 지연 취소 기능 (프로그래스바 포함)

즐겨찾기 별 모양을 브라우저 환경에 구애받지 않도록 `★` 아이콘으로 통일하고, `FAV` 및 `FAV2` 탭 내에서 즐겨찾기를 수정/해제할 때 실수 방지를 위한 5초 지연 및 취소(Undo) 기능과 얇은 그래디언트 프로그래스바를 테이블 헤더 바로 아래에 추가합니다.

## 사용자 검토 필요 사항
> [!IMPORTANT]
> - 모든 별 아이콘 모양은 `★` (솔리드 스타) 기호로 통일되며, 노란색(`var(--accent)`), 파란색(`#3b82f6`), 빈 별(`☆`, 회색)의 색상 및 테두리로 구분됩니다.
> - 즐겨찾기 해제/변경 시 5초(5000ms)의 지연 시간이 적용되며, 테이블 헤더 바로 아래에 얇은 그래디언트 프로그래스바가 실시간으로 줄어드는 효과가 표시됩니다. 또한 별 옆에 작게 "취소" 버튼이 생성됩니다.

  - 현재 활성화된 탭이 `FAV` 또는 `FAV2`인 경우, `localStorage`에 즉시 반영하는 대신 5초 타이머(`setTimeout`)를 등록하고 대기열(`store.pendingFavActions`)에 추가한 뒤 해당 행을 즉시 업데이트하여 취소 버튼을 노출합니다.

- `window.cancelFavoriteChange(symbol, event)` 함수를 새롭게 생성하여, "취소" 버튼을 눌렀을 때 진행 중인 타이머를 해제하고 원본 행 디자인으로 원복합니다.
- 5초가 경과되어 정상 커밋 시 호출될 `commitFavoriteChange(symbol)` 함수를 추가하여 `localStorage` 반영 및 테이블 갱신을 실행합니다.
- 진행률 표시를 위한 `updateProgressBar()` 함수를 구현하여 실시간(`setInterval` 50ms 간격)으로 프로그래스바의 너비(100% -> 0%)를 업데이트합니다.

## 검증 계획

### 수동 검증
1. `ALL` 탭에서 빈 별 클릭 시 지연 없이 즉시 노란별로 변경 확인.
2. `ALL` 탭에서 노란별 클릭 시 지연 없이 즉시 파란별로 변경 확인.
3. `FAV` 탭으로 전환 후, 노란별 클릭 시:
   - 해당 코인 행의 별이 파란별로 미리 변경되고 옆에 "취소" 버튼 노출 확인.
   - 테이블 헤더 바로 아래에 얇은 그래디언트 프로그래스바가 100%에서 0%로 실시간 감소 시작 확인.
4. 5초가 지나기 전 "취소" 버튼 클릭 시:
   - 프로그래스바가 사라지고, 별이 다시 노란별로 돌아오며 "취소" 버튼이 사라지는지 확인.
5. "취소" 버튼을 누르지 않고 5초간 대기 시:
   - 5초 후 프로그래스바가 사라지고 해당 코인이 `FAV` 테이블에서 정상적으로 사라지는지(localStorage 커밋 완료) 확인.
6. `FAV2` 탭에서도 동일하게 파란별 클릭 시 5초 지연, 취소 버튼 표시, 프로그래스바 진행, 5초 후 삭제(커밋)가 정상 작동하는지 검증.
