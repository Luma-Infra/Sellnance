# 구현 계획서 - Custom 로그 스케일 Mcap 및 Volume 필터 드롭다운

"Hiding Mcap < 1M" 버튼 옆에 필터링 이모지가 포함된 "Custom" 필터 버튼을 추가합니다. 이 버튼을 클릭하면 **시가총액(Market Cap)** 및 **거래량(Volume)**에 대한 세로 정렬된 로그 스케일 범위 슬라이더와 업비트/바이낸스 볼륨 토글이 들어있는 드롭다운이 나타납니다.

## 사용자 검토 필요 사항
- 로그 스케일 슬라이더는 0부터 최대값(Mcap은 2T, Volume은 100B)까지 선택할 수 있으며, 10배 단위(1M, 10M, 100M, 1B 등)로 보조선과 틱이 배치됩니다.
- 볼륨 필터에는 바이낸스(Binance)와 업비트(Upbit) 데이터 중 필터링 기준으로 삼을 대상을 선택할 수 있는 토글 스위치가 제공됩니다.
- 드롭다운 내부에 모든 설정을 초기 범위로 되돌리는 "초기화(Reset)" 버튼이 포함됩니다.
- 슬라이더 값을 조절할 때 테이블 데이터가 실시간으로 필터링되어 반영됩니다.

## 변경 예정 사항

### 전역 상태 구성
#### [MODIFY] [_store.js](file:///c:/Users/kmj/Sellnance/static/_store.js)
- 전역 `store` 객체에 다음 상태 값을 추가하여 필터링 범위를 관리합니다:
  - `customFilterActive`: 커스텀 필터 활성화 여부 (boolean)
  - `customMcapMin`: 시가총액 최소값 (기본값: 0)
  - `customMcapMax`: 시가총액 최대값 (기본값: 2,000,000,000,000 / 2T)
  - `customVolMin`: 거래량 최소값 (기본값: 0)
  - `customVolMax`: 거래량 최대값 (기본값: 100,000,000,000 / 100B)
  - `customVolSource`: 거래량 기준 거래소 (기본값: `'BINANCE'`)

### 사용자 인터페이스 레이아웃
#### [MODIFY] [index.html](file:///c:/Users/kmj/Sellnance/templates/index.html)
- `#btn-small-cap` 버튼 옆에 `btn-custom-filter` 버튼을 배치합니다.
- 이 버튼 하단에 `#custom-filter-dropdown` 구조를 추가합니다.
- 스타일이 입혀진 이중 핸들(Dual-thumb) 슬라이더 인풋과 트랙 하이라이트 처리를 위한 CSS 스타일 선언을 추가합니다.

### 필터링 로직 구현
#### [MODIFY] [table_filter.js](file:///c:/Users/kmj/Sellnance/static/table_filter.js)
- `getFilteredData()`에 커스텀 범위 필터링 로직을 반영합니다:
  - 각 코인의 `MarketCap_Raw`가 `customMcapMin`과 `customMcapMax` 사이에 속하는지 확인합니다.
  - `customVolSource` 선택에 따라 코인의 `Volume` 또는 `VolumeUpbit` 값을 `customVolMin`과 `customVolMax` 범위와 비교합니다.
- 아래의 글로벌 이벤트 핸들러 및 유틸 함수들을 작성합니다:
  - `toggleCustomFilter(event)`: 드롭다운을 열고 닫으며, 클릭 시 타임프레임 드롭다운 등 다른 드롭다운을 닫습니다.
  - `setVolSource(source)`: 거래량 소스(BINANCE / UPBIT)를 토글하고 테이블을 갱신합니다.
  - `applyCustomFilter()`: 설정을 적용하고 드롭다운을 닫습니다.
  - `resetCustomFilter()`: 모든 슬라이더 범위를 기본값(전체 선택)으로 되돌리고 테이블을 갱신합니다.
  - 로그 스케일 변환 헬퍼 함수 (`sliderToMcap`, `mcapToSlider`, `sliderToVol`, `volToSlider`).

### 반응형 뷰 모드 처리
#### [MODIFY] [ui_control.js](file:///c:/Users/kmj/Sellnance/static/ui_control.js)
- `switchViewMode(mode)`에서 뷰 모드가 Simple이거나 모바일인 경우 버튼 내 "Custom" 글자를 숨기고 이모지(`🎛️`)만 노출되도록 제어합니다.

## 검증 계획
### 수동 검증
- 대시보드를 실행하고 새 "Custom" 버튼과 드롭다운을 활성화하여 동작을 확인합니다.
- 슬라이더를 부드럽게 움직이며 테이블 행들이 실시간으로 로그 스케일 기준에 맞춰 필터링되는지 확인합니다.
- 업비트와 바이낸스 거래량 토글을 변경하며 필터링 기준이 잘 바뀌는지 검증합니다.
- 초기화(Reset) 버튼을 클릭하여 슬라이더 값과 테이블 전체 상태가 복구되는지 검증합니다.
