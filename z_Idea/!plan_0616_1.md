# 바이낸스 주식 토큰(bStocks) 매칭 실패 원인 분석 및 해결 보고서

바이낸스 선물(Futures) 마켓의 주식 토큰(예: `SPCX`, `TSLA` 등)과 현물(Spot) 마켓의 토큰화 주식(예: `SPCXB`, `TSLAB` 등) 간의 데이터 불일치 및 매칭이 어긋나던 문제를 분석하고 코드로 완벽히 조치하였습니다.

---

## 1. 문제의 원인 분석

### ① 선물-현물 간의 심볼 불일치 (B-Suffix 문제)
* **선물(Futures):** 실제 주식 티커인 `SPCX`, `TSLA`, `SNDK`, `NVDA` 등을 그대로 사용합니다.
* **현물(Spot):** bStocks를 의미하는 접미사 `B`가 붙은 `SPCXB`, `TSLAB`, `SNDKB`, `NVDAB` 등으로 거래됩니다.
* **영향:** 기존 백엔드 코드가 이를 문자열로 단순 비교하여 **완전히 다른 자산**으로 처리해 대시보드상에 두 줄로 쪼개져 출력되었습니다. 이로 인해 베이시스(Basis) 연산 및 현물 시세/볼륨 통합이 불가능했습니다.

### ② CoinMarketCap(CMC)의 오맵핑 문제
* `mapping.json`에서 선물 심볼인 `SPCX`가 비활성 프로젝트이자 별개 프로젝트인 **"Paimon SpaceX SPV Token" (ID: 36921, 시총/가격 0)**으로 매핑되어 정보가 노출되지 않았습니다.
* 실제 바이낸스 현물에 상장되어 거래 중인 **"SpaceX Tokenized bStocks"**는 **ID `40217`**이었습니다.

### ③ 백엔드 서버의 mapping.json 주기적 오버라이트(Overwrite)
* 활성화된 백엔드 서버 프로세스가 백그라운드에서 주기적으로 `mapping.json` 파일을 업데이트(환율 최신화 등)하면서 직접 손으로 수정한 정적 데이터 구조를 무시하고 덮어써서 복구가 무력화되는 문제가 있었습니다.

---

## 2. 해결 방안 및 구현 내용

백그라운드 서버 프로세스가 살아있는 점을 감안하여, **자동 복구(Self-Healing) 방식**과 **로컬 하드코딩 폴백(Fallback) 방식**을 함께 도입하여 코드와 족보 데이터를 완벽하게 안전지대로 이전시켰습니다.

### ① 주식 토큰 현물-선물 매핑 오버라이드 구현
[modules/exchange_api.py](file:///c:/Users/kmj/Sellnance/modules/exchange_api.py#L514-L529)의 바이낸스 현물/선물 데이터 수집부에 `STOCK_SPOT_TO_FUTURES` 전처리 로직을 심었습니다.
* 백그라운드 서버의 파일 덮어쓰기 문제에 대비하여, JSON에서 읽어오지 못할 시 하드코딩된 주식 토큰 매핑 데이터셋으로 폴백하도록 조치했습니다.
* `ARB`나 `BB` 같은 일반 코인명이 잘못 변환되는 일이 없도록 등록된 타겟(`SPCXB` ➡️ `SPCX` 등)에 한해서만 변환을 적용했습니다.

### ② 자기 치유(Self-Healing)형 CMC ID 보정
[modules/builder_binance.py](file:///c:/Users/kmj/Sellnance/modules/builder_binance.py#L52-L57)에 `SPCX`가 CMC ID `36921`이 아닌 `40217` (SpaceX Tokenized bStocks)을 바라보도록 런타임에 강제 보정하는 코드를 작성했습니다.
* 런타임에 보정된 족보 데이터는 백엔드 서버의 자동 저장 기능(`save_mapping_data`)을 타고 **디스크의 `mapping.json` 파일에 영구적으로 안전하게 반영**됩니다.

## 3. 검증 결과

실제 시세 데이터를 조회하는 테스트 스크립트(`scratch/verify_merge.py`)를 가동해 데이터를 검증한 결과는 다음과 같습니다.

* **결과 로그:**
  ```text
  Target SPCXUSDT found in binance_data:
    price: 216.42
    change_24h: 29.353
    vol_futures: 2983408139.6031
    vol_spot: 27554580.85573         <-- 현물(SPCXB) 볼륨이 SPCX 선물로 정상 통합됨
    is_spot_only: False
    is_futures: True
    is_spot: True                    <-- Spot & Futures가 한 행으로 묶임
  
  SUCCESS: SPCXBUSDT successfully removed/merged from binance_data.
  ```

이로써 대시보드 상에서 **코인-주식 분류가 정확하게 영역 분리**되고, 선물/현물 시세와 거래량 및 베이시스가 단일 `SPCX` 행에 완벽하게 집계됩니다.
