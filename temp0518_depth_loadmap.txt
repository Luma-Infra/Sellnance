🏛 [최종 완성판] 쌀먹 최적화 HTS 호가창 로드맵 (Orderbook + Precision + Theme + Lifecycle)
🚀 단계 1: UI 레이아웃 및 동적 툴바 DOM 설계 (구조 뼈대)
우측 도킹 패널: 메인 차트 영역 우측에 가로 너비 w-[260px] 규모의 세로형 호가창 컨테이너(#orderbook-panel) 배치.
스마트 정밀도 툴바: 패널 상단에 store.getPrecision 결과값에 맞춰 실시간으로 버튼 라벨이 갈아끼워지는 동적 툴바 부착.
헤더 토글 버튼: 상단 네비게이션 바에 호가창 패널을 부드럽게 열고 닫을 수 있는 FLIP 기반 토글 버튼 연동.
🚀 단계 2: 바이낸스 실시간 호가 스트림 연동 및 🚨 [쌀먹 생명주기 관리]
단일 소켓 교체 엔진 (store.orderbookWs): streamEach.js의 차트 소켓 관리 기법을 100% 재사용합니다. 트레이더가 코인을 클릭할 때마다 아래의 철벽 방어 로직이 작동합니다.
기존 구독 완벽 절단: if (store.orderbookWs) store.orderbookWs.close();를 호출하여 이전 코인의 호가 스트림을 즉시 소멸시킵니다.
단일 신규 구독: 오직 현재 선택된 코인의 @depth20@100ms 스트림만 단독으로 신규 연결합니다.
효과: 동시에 수백 개의 유령 구독이 방치되어 바이낸스 서버로부터 IP 밴(Ban)을 당하거나 브라우저 메모리(RAM)가 폭발하는 사태를 10000% 원천 차단합니다.
렌더링 방어막: 초당 수십 번 날아오는 호가 데이터로 인한 브라우저 멈춤 현상을 막기 위해, requestAnimationFrame과 연동된 100ms 단위 스로틀링(Throttling) 파이프라인 구축.
🚀 단계 3: 초고속 실시간 정밀도 병합 엔진 (Precision Aggregation Engine)
트레이더가 툴바에서 선택한 병합 단위(예: 10달러)에 맞춰, 수신된 20호가의 매수/매도 잔량을 0.001초 만에 합산(Grouping)해주는 고속 연산 엔진 부착.
병합 공식: groupedPrice = (Math.floor(rawPrice / groupSize) * groupSize).toFixed(p)
병합된 각 호가창의 누적 잔량(Total Size)을 계산하여, 배경에 깔리는 뎁스 바(Depth Bar)의 가로 너비 비율을 실시간 산정.
🚀 단계 4: DOM 최적화 및 하드웨어 가속 렌더링 (60FPS 달성)
호가창이 바뀔 때마다 innerHTML을 통째로 지우고 새로 그리는 원시적 방식 원천 배제.
최초 1회 생성된 20줄의 DOM 노드를 그대로 유지한 채, 텍스트(textContent)와 막대 너비(style.width)만 갈아끼우는 단일 진실 공급원(Single Source of Truth) 업데이트 기법 적용.
will-change: width 속성과 GPU 가속을 통해 호가창이 미친 듯이 쏟아져도 깜박임(Flickering)이나 버벅임 없는 극강의 부드러움 달성.
🎨 🚀 단계 5: 라이트/다크 테마 실시간 동기화 아키텍처 (Theme Engine)
호가창 내부의 색상을 자바스크립트에서 하드코딩하지 않고, CSS 변수(var(--up), var(--down), var(--panel))와 100% 연동하는 반응형 테마 엔진을 구축합니다.

배경 및 테두리 연동: 호가창 컨테이너의 배경을 bg-theme-panel, 테두리를 border-theme-border로 지정하여 테마 토글 시 메인 차트 패널과 동일하게 0.3초 만에 스르르르륵 전환되도록 설정.
텍스트 및 가격 색상 연동: 매도 호가 가격은 text-theme-down, 매수 호가 가격은 text-theme-up 클래스를 부여하여 테마 전환 시 즉각 반응하도록 구성.
뎁스 바(Depth Bar) 반투명 처리: 뎁스 바 배경색을 bg-theme-down/10(매도), bg-theme-up/10(매수) 등 테일윈드 투명도 유틸리티로 지정하여 다크 모드와 라이트 모드 모두에서 완벽한 가독성 확보.
